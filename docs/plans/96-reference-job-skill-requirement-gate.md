# 参照ジョブ追加 Skill 要件確認ゲート整備計画

## ステータス

Done

## 目的

`add-reference-batch-job` Skill が、要件の不足した状態で実装を早めないようにする。

一方で、すべての依頼に詳細な要件定義を要求して作業を硬直化させず、影響度と可逆性に応じて、AI が次のいずれかを判断できる状態を目指す。

1. 要件が十分であり、そのまま実装を開始する
2. 低リスクな不明点を暫定前提として明示し、小さな試行を開始する
3. 重要な不明点を絞って確認し、実装を保留する

## 背景

現在の Skill は、既存の tasklet、DB chunk、file chunk の参照パターンに沿ったジョブ追加には利用できる。

ただし、新しいバッチパターンを検討する場合について、次の判断が十分に明文化されていない。

- どの程度の情報があれば実装を開始してよいか
- 不明点を仮定として扱ってよい範囲
- 実装前に確認が必要な不明点
- 既存 3 パターンに該当しない場合の進め方
- プロジェクトに要件が存在しないことを、AI が明確に伝える方法

この不足を、固定的なチェックリストではなく、判断を支援する軽量なゲートとして補う。

## 基本方針

### 1. 質問数ではなく影響度で判断する

要件項目を毎回すべて質問しない。

不明点が次の領域へ影響する場合に限り、実装前の確認を優先する。

- データの重複、欠損、不整合
- トランザクション境界
- restart、rerun、冪等性
- skip、retry、rollback の結果
- 入出力の確定または公開タイミング
- 並行実行と排他
- 機密情報、監査、権限制御
- 運用終了コードや障害復旧

名称、サンプル値、軽微な形式など、後から容易に変更できる事項は、仮定を明示して進められる。

### 2. 三つの進行モードを使う

#### 実装開始

ジョブの目的、主要な入出力、正常終了条件、失敗時の扱い、再実行方針が判断できる場合に選択する。

#### 暫定試行

不明点が低リスクかつ可逆であり、参照実装によって設計判断を検証できる場合に選択する。

AI は重要な暫定前提と試行の対象外を短く示し、実装範囲を最小化する。

#### 設計確認

不明点がデータ整合性、セキュリティ、監査、復旧方式、共通基盤の責務へ影響する場合に選択する。

AI は網羅的な質問票を提示せず、実装判断を変える質問を原則 1 から 3 件に絞る。

### 3. 影響度と可逆性から進行モードを選ぶ

次の順序を判断の目安とし、項目数による機械的な採点には使用しない。

1. 不明点が、データ整合性、復旧、セキュリティ、監査、運用契約、共通基盤の責務へ影響するか確認する。
2. 影響する場合は `設計確認` とし、判断に必要な質問だけを提示する。
3. 影響が限定的で、変更を局所的に戻せる場合は `暫定試行` とし、暫定前提と対象外を明示する。
4. 主要な振る舞いを既存要件と実装から判断できる場合は `実装開始` とする。

情報が少ないこと自体を停止理由にはしない。反対に、情報が多くても重要な判断が未確定なら実装を開始しない。

### 4. 既存パターンへ無理に適合させない

tasklet、DB chunk、file chunk のいずれにも自然に該当しない場合は、次を区別する。

- 既存パターンの組み合わせで表現できる
- 小さな設計検証で適合性を確認できる
- 新しい共通機能または新しいジョブパターンが必要である

新しいパターンが必要な可能性がある場合は、完成形を先行実装せず、設計上の未決事項と最小の検証内容を提示する。

### 5. 要件の所在を明確にする

リポジトリ文書や既存コードに要件がない場合、AI は推測を既定仕様として扱わない。

次を区別して応答する。

- 確認できた事実
- 作業上置く仮定
- 利用者または業務設計で決定が必要な事項

- `docs/agent/skills/add-reference-batch-job/SKILL.md`

## 対象外

- 新しいバッチジョブの実装
- 新しいフレームワーク機能の実装
- 全業務で共通利用する詳細な要件定義テンプレート
- AI ツール固有の設定ファイル
- すべての設計文書の全面改訂

## 完了条件

- Skill に三つの進行モードと選択基準が記載されている
- 重要な不明点と、仮定して進められる不明点が区別されている
- 既存 3 パターンに適合しない場合の分岐が記載されている
- AI が事実、仮定、要決定事項を区別して応答できる
- 質問は実装判断に必要な少数へ絞る方針が記載されている
- 代表シナリオで、過早な実装と過剰な停止の双方を避けられることを確認している
- Skill が参照する文書に、現状と明確に矛盾する記述が残っていない
